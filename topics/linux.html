Basic Git Commands and Usage

Added: 2025-12-03 12:17

This document provides a comprehensive overview of essential Git commands for managing repositories, branches, and changes. It covers basic operations such as initializing a repository, staging changes, and viewing commit history, as well as more advanced features like branching and stashing.

Key Points

  • Git is a version control system that helps manage code changes.
  • Basic commands include initializing repositories, staging, committing, and pushing changes.
  • Branching allows for parallel development and easier collaboration.

Commands

CommandDescription
git init Initializes a new Git repository in the current directory.
git clone <repo_url> Clones a repository from a remote URL to your local machine.
git status Shows the working directory status (which files are modified, staged, etc.).
git add <file> Stages a file or files to be committed.
git commit -m "message" Commits staged changes to the local repository with a message.
git push Pushes committed changes to the remote repository.
git pull Fetches and merges changes from the remote repository to your local working directory.
git branch Lists all branches in your repository.
git checkout <branch_name> Switches to an existing branch.
git merge <branch_name> Merges changes from the specified branch into the current branch.
git reset <commit_id> Moves HEAD to a specific commit, leaving your working directory intact.
git stash Stashes the changes in your working directory (both staged and unstaged).
Regularly commit changes to keep track of your progress.
Use branches for new features or bug fixes to avoid disrupting the main codebase.

‚Ä¢ Write clear commit messages to describe the changes made.

‚Ä¢ Pull changes from the remote repository frequently to stay updated with team progress.

Git Stash, Tags, Config, Aliases, Hooks, Submodules, Workflow Commands, Merge Strategies, Bisect, Clean and Prune

Added: 2025-12-03 12:17

This section covers various Git commands and concepts including stashing changes, managing tags, configuring Git settings, creating aliases, using hooks for automation, handling submodules, collaborating with others through workflow commands, resolving merge conflicts, finding bugs with bisect, and cleaning up repositories.

Key Points

  • Stashing allows you to save changes temporarily without committing them.
  • Tags are used to mark specific points in the repository's history.
  • Git hooks automate tasks during Git actions.

Commands

CommandDescription
git stash pop Applies the latest stash and removes it from the stash list.
git stash apply Applies a stash without removing it from the list.
git stash drop Removes a specific stash from the stash list.
git stash clear Clears all stashes.
git stash save "message" Stashes changes with an optional message for identification.
git tag Lists all tags in the repository.
git tag <tag_name> Creates a new tag at the current commit.
git config --global user.name "Your Name" Sets your global username for commits.
git config --global user.email "youremail@example.com" Sets your global email for commits.
git pull --rebase Fetches and applies remote changes, rebasing your local commits on top of them.
git merge --no-ff Merges a branch with a 'no fast-forward' option.
git bisect start Starts a binary search for the commit that introduced a bug.
git clean -f Removes untracked files from the working directory.
git gc Runs garbage collection, cleaning up unnecessary files.
Use descriptive messages when stashing changes for easier identification later.
Regularly clean up your repository to maintain performance.

‚Ä¢ Always pull changes before starting new work to avoid conflicts.

‚Ä¢ Use tags to mark release points in your project for easy reference.

Git Commands for DevOps and CI/CD

Added: 2025-12-03 12:17

This section covers essential Git commands and practices for DevOps, focusing on Continuous Integration/Continuous Deployment (CI/CD), Infrastructure as Code (IaC), and automation through Git hooks. It highlights how Git can be utilized for managing code, tracking changes, and deploying applications across multiple environments.

Key Points

  • Git commands are crucial for CI/CD pipelines and managing Infrastructure as Code.
  • Git hooks automate tasks within the DevOps pipeline, ensuring code quality and stability.
  • Versioning and tagging in Git are essential for consistent deployments across environments.

Code Examples

Creates a tarball archive of the repository at a specific branch.

git archive --format=tar --output=<output_file>.tar <branch_name>

Creates a zip archive of the repository at a specific branch.

git archive --format=zip --output=<output_file>.zip <branch_name>

Fetches all branches from all remotes.

git fetch --all

Merges the latest changes from a remote branch into the local branch.

git merge origin/

Clones repositories that contain configuration files or infrastructure code.

git clone <IaC_repo_url>

Switches to a branch that contains a specific environment‚Äôs infrastructure code.

git checkout <branch_name>

Pulls the latest changes to infrastructure code before applying configurations.

git pull origin <branch_name>

Creates a branch dedicated to developing a specific feature.

git branch <feature_flag_name>

Merges a completed feature flag branch into the main codebase.

git merge <feature_flag_branch>

Triggers CI/CD pipelines that automatically deploy code to specific environments.

git push origin <branch_name>

Commands

CommandDescription
git blame <file_path> Shows who last modified each line of a file and when.
git annotate <file_path> Displays commit information for each line of a file.
git reflog Displays a log of all movements of the HEAD pointer.
git reflog show Displays a detailed view of HEAD and the state of the repository.
git reflog expire --expire=now --all Removes all reflog entries that have expired.
git reflog delete <ref_id> Deletes a specific reflog entry.
git push --force-with-lease Pushes changes forcefully while ensuring the remote branch hasn‚Äôt changed unexpectedly.
git rebase origin/ Rebases your branch to ensure it is up to date with the base branch.
git tag <version_number> Tags releases for versioning in CI/CD.
git merge --no-ff <feature_branch> Merges feature branches into specific environment branches with a merge commit.
git reset --hard Resets the working directory to a known state before deploying.
Use Git hooks to automate testing and deployment processes.
Regularly tag releases to maintain a clear versioning history.

‚Ä¢ Always pull the latest changes before deploying to ensure you are working with the most recent code.

‚Ä¢ Use descriptive commit messages and enforce conventions through Git hooks.

Advanced Git for Managing CI/CD Pipelines

Added: 2025-12-03 12:17

This section discusses how Git integrates with CI/CD pipelines and Docker workflows, highlighting commands and practices that enhance deployment processes. It also covers secrets management and commit history management using Git.

Key Points

  • GitOps automates deployment pipelines through Git repository changes.
  • Git commands facilitate CI/CD processes and Docker image management.
  • Secrets management and commit history cleanup are crucial in DevOps.

Code Examples

Clone a repository containing Dockerfile and application code

git clone <repo_url>

Switch to a specific branch

git checkout <branch_name>

Pull the latest code from a branch

git pull origin <branch_name>

Remove a file from version control

git rm --cached <file>

Amend the last commit

git commit --amend

Start an interactive rebase

git rebase -i <commit_id>

Sign a commit with your GPG key

git commit --gpg-sign

Commands

CommandDescription
git push Triggers deployment pipelines by pushing changes to a Git repository.
git push --tags Triggers a CI/CD pipeline that looks for tags to start a deployment.
git fetch --tags Fetches the latest tags in CI/CD pipelines.
git log --oneline --graph Visualizes commit histories during pipeline execution.
git diff <commit_id> <commit_id> Shows changes between commits to decide on triggering builds.
git rebase --onto Rebases a branch onto a new base commit.
git merge --abort Aborts the merge process and restores the previous state.
git config --global user.signingkey <key_id> Configures the GPG key for signing commits and tags.
git tag -s <tag_name> Creates a signed tag for marking important release points.
Use .gitignore to keep sensitive files like secrets out of version control.
Regularly clean up commit history using rebase to maintain a linear history.

‚Ä¢ Integrate Git with CI/CD tools for automated deployments.

‚Ä¢ Use Git tags for versioning Docker images and releases.

This section covers essential Git commands and practices for managing continuous delivery and rollbacks in DevOps. It highlights how Git integrates with automated release pipelines and the importance of reverting changes when necessary.

Key Points

  • Rolling back changes is crucial for maintaining stable production environments.
  • Managing dependencies with Git submodules and subtrees enhances collaboration.

Commands

CommandDescription

git checkout <commit_id> Checks out an older commit.
git submodule update --recursive Updates all submodules.

git push origin <feature_branch> Pushes a feature branch to a remote.
git fetch origin <feature_branch> Fetches a feature branch from a remote repository.
git pull origin <feature_branch> Pulls the latest changes from a feature branch.
git diff <branch_name> Compares different branches before merging.
git clone <repo_url> Clones a repository containing code and configurations.
git checkout Switches between branches.
git pull origin <branch_name> Pulls the latest code from a Git repository.
git tag Tags releases to facilitate versioning.
Use descriptive commit messages to make rollbacks easier.
Regularly update submodules to ensure all dependencies are in sync.

  • Versioning of configuration files and documentation is essential in DevOps.
  • Safe force pushing helps maintain collaboration in CI/CD workflows.

Commands

CommandDescription
git push origin Pushes your branch to the remote repository, triggering the build process in your CI/CD tool.
git tag <build_version> Tags a commit with a version number, which might trigger deployment pipelines for testing or production.
git log --oneline View a condensed log of the commit history to identify the changes associated with a specific build or deployment.
git push --force-with-lease Pushes changes to a branch with a safer force push, useful when you need to overwrite history in collaborative CI/CD workflows without overwriting someone else‚Äôs work.
git add Add configuration or documentation files to the repository for versioning.
git commit -m "add/update documentation" Commit changes related to documentation, often used in documentation-driven development.
git diff Compare different versions of documentation files or configuration files, useful when tracking changes in infrastructure or environment files.
Use tags to manage different versions of your builds effectively.
Regularly commit documentation changes to keep track of updates.

‚Ä¢ Always review commit history before pushing changes to avoid conflicts.

‚Ä¢ Utilize safe force pushing to prevent overwriting others' work in collaborative environments.
